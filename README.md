# Note Mark API

A RESTful API for personal markdown note-taking with JWT authentication, PostgreSQL persistence, and grammar checking.

Built as a solution to the [Markdown Note-taking App](https://roadmap.sh/projects/markdown-note-taking-app) challenge from [roadmap.sh](https://roadmap.sh).

## Features

- **JWT Authentication** — register, login, and protect all endpoints with Bearer tokens
- **User-scoped notes** — each user's notes are completely isolated from other users
- **Markdown CRUD** — create, read, update, delete, and upload `.md` files
- **HTML rendering** — render note content to HTML with syntax highlighting support
- **Grammar checking** — powered by the LanguageTool API
- **PostgreSQL** — production-ready database with SQLAlchemy ORM
- **Docker Compose** — one-command local environment with API + database

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | FastAPI |
| ORM | SQLAlchemy 2.x |
| Database | PostgreSQL 16 |
| Auth | JWT (python-jose) + bcrypt |
| Markdown | Python-Markdown |
| Grammar | LanguageTool API |
| Tests | pytest + httpx |
| Container | Docker / Docker Compose |

## Quick Start

### With Docker Compose (recommended)

```bash
# Clone the repository
git clone <repository-url>
cd notes-api

# Start API + PostgreSQL
docker compose up --build

# API is available at http://localhost:8000
```

### Without Docker

```bash
# 1. Create and activate a virtual environment
python -m venv venv
source venv/bin/activate   # Linux / macOS
# venv\Scripts\activate    # Windows

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure environment variables
cp .env.example .env
# Edit .env and fill in DATABASE_URL and SECRET_KEY

# 4. Run the server
uvicorn app.main:app --reload
```

## Environment Variables

Copy `.env.example` to `.env` and configure:

| Variable | Description | Example |
|---|---|---|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://user:pass@localhost:5432/notesdb` |
| `SECRET_KEY` | JWT signing key (use a long random string) | `openssl rand -hex 32` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Token TTL in minutes (default: 30) | `60` |

## Endpoints

### Auth

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/auth/register` | Create a new user account |
| `POST` | `/auth/token` | Login and receive a JWT token |

### Notes (requires Bearer token)

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/notes/` | Create a new note |
| `GET` | `/notes/` | List all notes for the authenticated user |
| `GET` | `/notes/{id}` | Get a note by ID |
| `GET` | `/notes/{id}/html` | Get note rendered as HTML |
| `PUT` | `/notes/{id}` | Update a note |
| `DELETE` | `/notes/{id}` | Delete a note |
| `POST` | `/notes/upload` | Upload a `.md` file as a note |

### Grammar (requires Bearer token)

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/grammar/check` | Check text for grammar and spelling errors |

## Usage Examples

### Register and login

```bash
# Register
curl -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username": "alice", "email": "alice@example.com", "password": "mypassword"}'

# Login — returns access_token
curl -X POST http://localhost:8000/auth/token \
  -F "username=alice" \
  -F "password=mypassword"
```

### Create and read notes

```bash
TOKEN="your_access_token_here"

# Create a note
curl -X POST http://localhost:8000/notes/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title": "My Note", "content": "# Hello\n\nThis is **bold**."}'

# List notes
curl http://localhost:8000/notes/ \
  -H "Authorization: Bearer $TOKEN"

# Get note as HTML
curl http://localhost:8000/notes/1/html \
  -H "Authorization: Bearer $TOKEN"
```

### Upload a markdown file

```bash
curl -X POST http://localhost:8000/notes/upload \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@mynote.md"
```

### Check grammar

```bash
curl -X POST http://localhost:8000/grammar/check \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"text": "This is a test sentnce with erors."}'
```

## Running Tests

Tests use an in-memory SQLite database for full isolation — no external services required.

```bash
pip install -r requirements.txt
pytest tests/ -v
```

Expected output: **26 tests passing** across `test_auth`, `test_notes`, and `test_grammar`.
